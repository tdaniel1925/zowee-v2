Animation States

Playing

Messages appear sequentially with delays. Typing indicator shows before each incoming message.

Paused

Animation halts at current message. All timers suspended. Resume resumes from exact position.

Typing

Three-dot bounce indicator shown before incoming messages. Duration configurable via prop.

Cycling

After last message, fades out and restarts next scenario. Loop count configurable.

Delay Badges

Shown between messages to indicate timing gaps in the conversation flow.

+0.5s delay +1.2s delay +2.0s delay +3.5s delay
custom via delay prop

Component Props

TypeScript
Prop Type Default Description
scenarios Scenario[] required Array of conversation scenarios to cycle through
autoPlay boolean true Start animation automatically on mount
typingDuration number 1200 Duration in ms to show typing indicator
messageDelay number 800 Base delay between messages in ms
loop boolean true Loop through all scenarios continuously
fadeTransition number 400 Fade transition duration between scenarios in ms
showDelayBadges boolean true Show timing delay badges between messages
onScenarioChange function undefined Callback fired when scenario changes

Usage Example

AnimatedSmsDemo.tsx 
import { AnimatedSmsDemo } from '@zowee/components';

const scenarios = [
  {
    id: 'welcome',
    messages: [
      { type: 'incoming', text: 'Welcome to ZOWEE! 🎉', delay: 0 },
      { type: 'outgoing', text: 'Thanks! How does it work?', delay: 1200 },
      { type: 'incoming', text: 'Your rep earns on every sub.', delay: 800 },
    ]
  },
  // ... more scenarios
];

export default function MyPage() {
  return (
    <AnimatedSmsDemo
      scenarios={scenarios}
      autoPlay={true}
      typingDuration={1200}
      messageDelay={800}
      loop={true}
      fadeTransition={400}
      showDelayBadges={true}
      onScenarioChange={(id) => console.log('Scenario:', id)}
    />
  );
}
